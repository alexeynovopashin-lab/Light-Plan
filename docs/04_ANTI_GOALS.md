# What We Will Never Become

Мы никогда не будем:

## Социальной сетью

Фотограф приходит принимать решения.
Не публиковать контент.

## Приложением ради вовлечения

Мы не измеряем успех количеством минут внутри приложения.
Наш идеальный сценарий — пользователь получил ответ и сразу закрыл приложение.

## Копией Sun Surveyor

Мы не соревнуемся количеством функций.
Мы соревнуемся качеством решений.

## Энциклопедией астрономии

Мы используем сложные модели.
Но никогда не заставляем пользователя изучать их.

## Панелью приборов

Мы не строим dashboard.
Мы строим опыт принятия решений.

## Каталогом функций

Каждая новая возможность обязана делать продукт проще.
Сложность считается не по приложению, а по экрану.
Функция не должна увеличивать сложность того экрана, где появляется.
Приложение целиком может знать больше.
Конкретный фотограф должен видеть меньше.

Отсюда второе требование.
Каждая новая возможность обязана уметь исчезнуть.
Если функцию нельзя убрать с глаз — она либо обязательна для всех, либо не нужна никому.

Убирать с глаз мы умеем тремя способами.
Режим: простой и инженерный.
Жанр: стрит-фотографу нечего сдавать, и срока сдачи у него нет.
Уровень: подробности живут глубже и открываются по желанию.
Ни один из трёх не спрашивает пользователя, кто он такой.
Он выбирает съёмку, а не себя.

У правила есть одна оговорка.
Планировщик добавляет функции сознательно: клиенты, сроки сдачи, деньги.
Это разрешено только внутри «Съемок» и только потому, что избавляет фотографа от отдельных приложений.
На экраны света оговорка не распространяется никогда.

## Планировщиком, который протек на главный экран

Экран «Сегодня» отвечает только за свет.
Ни одна задача, ни один клиент, ни один срок не должны на нем появиться.
Если планировщик начинает диктовать главный экран — граница нарушена.

## Продуктом, который следует моде

Мы не проектируем интерфейс под тренды года.
Мы проектируем продукт, который будет актуален через десять лет.

## Приложением, которое требует обучения

Хороший фотограф должен понимать интерфейс интуитивно.
Не читая руководство.

## Инструментом только для экспертов

Профессиональная глубина не должна превращаться в профессиональную закрытость.

## Коллекцией красивых экранов

Красивый экран ничего не стоит, если он не помогает принять решение.

## Продуктом без характера

Мы не копируем Apple.
Не копируем Leica.
Не копируем Halide.
Мы создаем собственый язык продукта.
Через несколько лет пользователь должен узнавать Light Plan без логотипа.

## Финальный принцип

Каждый раз, когда появляется новая идея, команда сначала открывает этот документ.
Если идея нарушает хотя бы один из перечисленных принципов, она не должна попасть в продукт без очень серьезного основания.
Характер продукта важнее количества функций.
