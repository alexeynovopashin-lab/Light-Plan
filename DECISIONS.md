# Light Plan — принятые решения

Документ нужен, чтобы не переспорить заново то, что уже решено, и не наступить
на грабли, которые уже разложены. Основания — в `.docx` (North Star, Principles,
UX Principles, Visual Language, Design System, Screen Specification, Anti-Goals).

---

## Разделение продукта

**«Сегодня» и «Карта» — тихая гавань, спокойный прибор света.
Весь CRM живёт внутри «Съёмок».**

Половина планов (расходы, сроки сдачи, поздравления, прибыль) — это CRM, а не свет.
Anti-Goals прямо запрещает превращаться в «каталог функций». Разделение защищает
главный экран: он остаётся про один вопрос — какой свет и когда.

**Зачем «Съёмки», если есть календарь телефона.** Стандартный календарь знает
«когда». Он не знает двух вещей: (1) будет ли свет — попадает ли съёмка в золотой
час и не испортился ли прогноз; (2) что ты должен — сроки сдачи материала.
Поэтому экран не должен открываться голой сеткой месяца.

## Сложность считается по экрану

Прототип показывают живым фотографам, и функций в нём пока меньше замысла, а не
больше. Прежняя формулировка Anti-Goals — «если функция увеличивает сложность,
она не появляется» — считала сложность по всему приложению и запрещала расти
вообще. Считаем по экрану: приложение может знать больше, конкретный фотограф
должен видеть меньше. Взамен появилось требование строже прежнего — **функция
обязана уметь исчезнуть**. То, что нельзя убрать с глаз, либо обязательно для
всех, либо не нужно никому. Запрет протечки планировщика на экраны света не
тронут.

**Жанр — пресет, а не профиль фотографа.** Профиль требует один раз объявить,
кто ты, и ошибается на смешанной практике: свадебщик снимает и семью тоже.
Пресет перестраивает форму под выбранный жанр (стриту не нужен блок сдачи
материала, свадьбе он главный), а ручные правки внутри жанра запоминаются.
Хранения на сервере не требует ни то, ни другое — всё лежит в `localStorage`.

## Латунный прибор, а не веб-форма

Чипсы — родом из веб-форм, самое «цифровое» место прототипа. Visual Language
прямо называет материалы продукта: матовый металл, полированная латунь,
измерительный прибор. Поэтому выборы в форме — инструменты, а не таблетки
с текстом:

- **жанры и погода** — пиктограммы с подписью, как кнопки режимов у камеры;
- **срок сдачи** — шкала с детентами и гравировкой делений; крайние деления —
  состояния («по жанру», «без срока»), середина — ручной срок;
- **ползунок времени** — риски каждый час, крупные каждые шесть; вместо слов
  «восход» и «закат» — пиктограммы со стрелкой, в лунном режиме кромки
  показывают восход и заход луны.

Граница: прибор — это поведение (шкалы, детенты, гравировка), а не имитация
материала. Градиенты «под латунь», блики и заклёпки — скевоморфизм, он
стареет плохо и запрещён духом Visual Language («не следуем моде»).

## Визуальный язык

**Красный заменён терракотой** (`#C9663D`). Visual Language: «избегаем красного —
показываем осторожность, а не опасность».

**Мигание заменено затуханием** (3.6 с, отключается при `prefers-reduced-motion`).
UX Principles, Calm Interaction: «мы не используем мигающие элементы». Медленное
дыхание цепляет периферийное зрение, но не дёргает.

**Срочность сдачи — непрерывная шкала, а не состояния**: белое → жёлтое →
янтарное → терракота по мере приближения к сроку. Свет в продукте тоже
непрерывен, дискретные «горит/не горит» выбивались бы из языка.

**Подложка, а не кольцо.** Тонкая окружность вокруг двузначного числа требует
оптической компенсации и всегда кажется кривой. Заливка этой проблемы лишена.

**Точка под датой = съёмка.** Нет съёмки — нет точки; чистый календарь абсолютно
пуст. Цвет точки говорит, какой свет достанется этой съёмке. Свет для планирования
смотрится выбором даты и сводкой под календарём.

**Кнопка — часть композиции, а не плашка-CTA** (Design System). Действие задаётся
размером и положением, а не насыщенным цветом.

**Контекстное действие вместо «+ Добавить».** Обычный календарь говорит «добавить
событие», наш — «добавить на 19:53, потому что там золотой час». Голая надпись
обесценивает продукт. Отдельная строка «нет событий» не нужна: пустой список сам
означает отсутствие.

**Шрифты — системный стек Apple.** `-apple-system` (SF Pro) для текста,
`ui-monospace` для чисел. Design System запрещает сторонние шрифты, и это честнее
для iPhone, чем веб-шрифты.

## Честность модели

Прогноз показывает **уверенность** и падает со временем («уверенность 70%,
уточнится ближе к дате») — принцип Trust.

**Белые ночи.** На 56° с.ш. летом солнце не опускается до −18°, астрономической
ночи нет. Модель это знает и честно говорит «белая ночь · звёзд не будет» вместо
бодрой цифры. Плохая погода перебивает лучший свет — иначе сводка врёт.

**Запланировать съёмку можно в любой день**, включая пасмурный: погода — не повод
запрещать запись клиента.

## Практические грабли

**Название.** Продукт называется **Light Plan** — везде: документы, заголовок
страницы, тексты внутри приложения. Прежнее имя Light Atlas выведено из
употребления, подмена заголовка при выкладке больше не нужна:

```bash
cp "<локальный>/index.html" "<репо>/index.html"
git add index.html && git commit && git push origin main
```

**Локальный файл не под git**: `/Users/alexey/Documents/Light Atlas/Light Atlas 2/index.html`.
Репозиторий клонируется отдельно.

**Кэш.** GitHub Pages и Safari агрессивно кэшируют. При проверке добавлять `?v=N`,
на телефоне — hard-refresh.

**Хранилище**: `localStorage`, ключ `lightplan.v1` (съёмки, настройки сдачи,
жанры, синхронизация). Даты сериализуются в ISO и восстанавливаются в `Date`.

**Флекс-ловушки, уже пойманные:**
- `width: 100%` вместе с боковыми `margin` даёт переполнение — ширину не задавать
- элементы в колоночном флексе сжимаются: критичным блокам нужен `flex-shrink: 0`
- `z-index: -1` у псевдоэлемента без своего контекста наложения уводит его за фон
- свёрнутый блок с `max-height: 0` всё равно занимает высоту паддинга —
  паддинг переносить внутрь
- правило ниже по файлу перебивает свойство при равной специфичности

**Что нельзя проверить в вебе:** тактильный отклик на iOS (Taptic из веба
недоступен, поэтому щелчки звуковые), настоящие пуш-уведомления, выгрузка в
облако. В прототипе это рисуется и имитируется.
